Git & GitHub Reference Guide

This guide compiles essential Git commands and workflows from multiple sources, presented in simple, clear steps. Use these commands directly in your terminal.

1. Installation & Setup

Installation

  • GitHub for Windows: https://windows.github.com
  • GitHub for Mac: https://mac.github.com
  • Git for All Platforms: http://git-scm.com

Configuration

  • git config --global user.name "Your Name" – Set your name for commit credits.
  • git config --global user.email "email@example.com" – Set your email for commit association.
  • git config --global color.ui auto – Enable command line coloring.

Initialize & Clone

  • git init – Initialize a Git repository in the current directory.
  • git clone [url] – Clone a repository from a URL (e.g., git clone https://github.com/user/repo.git).

2. Basic Commands

Stage & Snapshot

  • git status – Show modified files in the working directory.
  • git add [file] – Stage a specific file.
  • git add . – Stage all changes in the current directory.
  • git reset [file] – Unstage a file, keeping changes in the working directory.
  • git diff – Show unstaged changes.
  • git diff --staged – Show staged changes not yet committed.
  • git commit -m "message" – Commit staged changes with a message.
  • git commit -m "message" --amend – Update the previous commit.

Inspect & Compare

  • git log – Show commit history.
  • git log --all – Show all commits across branches.
  • git log --all --graph – Visualize branching history.
  • git log branchB..branchA – Show commits in branchA not in branchB.
  • git log --follow [file] – Show commit history for a file, including renames.
  • git diff branchB...branchA – Compare differences between branches.
  • git show [SHA] – Display details of a specific commit.

3. Branching & Merging

Branch Commands

Create, List, and Delete Branches

  • git branch – List all local branches (active branch marked with *).
  • git branch – Create a new branch.
  • git branch -a – List all branches (local & remote).
  • git branch -r – List only remote branches.
  • git branch -d – Delete a branch (safe deletion).
  • git branch -D – Force delete a branch.
  • git branch --move – Rename a branch.
  • git show-branch – Show branch history.
  • git checkout -b – Create and switch to a new branch.

Switch Between Branches

  • git checkout – Switch to a branch (older method).
  • git switch – Switch branch (recommended).
  • git switch -c – Create and switch to a new branch.

Merge Commands

Merging Branches

  • git merge – Merge specified branch into the current branch.
  • git merge --no-ff – Merge with a commit (no fast-forward).
  • git merge --abort – Abort a merge if conflicts arise.
  • git merge --squash – Squash multiple commits into one before merging.
  • git merge -m "message" – Merge with a commit message.
💡 Merge Conflict Resolution: Edit conflicting files to keep desired code, then run git add . and git commit -m "Resolve merge conflict". 🚀

4. Remote Repositories

  • git remote add [alias] [url] – Add a remote repository (e.g., git remote add origin https://github.com/user/repo.git).
  • git remote – List remote repositories.
  • git remote -v – List remotes with URLs (verbose mode).
  • git remote remove [alias] – Remove a remote.
  • git fetch [alias] – Fetch all branches from a remote.
  • git branch --track – Track a remote branch locally.
  • git fetch --all – Fetch all remote branches.
  • git pull origin – Pull latest changes from a remote branch.
  • git pull origin main --allow-unrelated-histories – Pull from main, allowing unrelated histories to merge.
  • git push [alias] [branch] – Push a branch to a remote.
  • git push [alias] [branch] --set-upstream – Push and set upstream tracking.
  • git push [alias] [branch] -f – Force push (overwrites remote).
  • git push origin --delete – Delete a remote branch.
  • git pull – Fetch and merge from the tracking remote branch.

5. Advanced Commands

Tracking Path Changes

  • git rm [file] – Remove a file and stage the deletion.
  • git mv [existing-path] [new-path] – Rename/move a file and stage it.
  • git log --stat -M – Show commit logs with moved paths.

Rewrite History

  • git rebase – Reapply commits on top of another branch.
  • git rebase --abort – Abort a rebase.
  • git rebase --continue – Continue after resolving conflicts.
  • git rebase -i HEAD~ – Interactive rebase to modify commit history.
  • git reset --hard [commit] – Reset to a commit, discarding changes.

Temporary Commits

  • git stash – Stash modified tracked files.
  • git stash list – List stashed changes.
  • git stash pop – Apply and remove the latest stash.
  • git stash drop – Discard the latest stash.

Handling Merge Conflicts

Identify Conflicts

  • git status – Check for merge conflicts.
  • git diff – Show differences in conflicting files.
  • git log --merge – Show commits causing conflicts.

Resolve Conflicts

  • Manually edit conflicting files to fix markers (<<<<<<<, =======, >>>>>>>)
  • git add – Mark conflict as resolved.
  • git commit – Commit the resolution.

Abort or Reset Merge

  • git merge --abort – Cancel the merge.
  • git reset --hard HEAD – Reset to last committed state.

Advanced Conflict Resolution

  • git checkout --ours – Keep changes from current branch.
  • git checkout --theirs – Keep changes from merged branch.
  • git reset --merge – Undo merge but keep local changes.

Other Utilities

  • git checkout -- [file] – Discard changes in a file.
  • git checkout [hash|branch] [file] – Restore a file from a commit.
  • git config --global alias.shortcut "command" – Create a command alias (e.g., git config --global alias.s "status").
  • rm -rf .git – Remove Git from a project.

Ignoring Files

Create a .gitignore file with patterns like: logs/, *.notes, pattern*/

  • git config --global core.excludesfile [file] – Set a global ignore file.

6. Feature Branch Workflow

  1. Create a Feature Branch:
    git branch new-feature
    git checkout new-feature
  2. Make Changes & Commit:
    git add .
    git commit -m "new feature message"
  3. Push to GitHub:
    git push origin new-feature
  4. Create a Pull Request: On GitHub, open a pull request for review.
  5. Merge into Main: After approval, merge via GitHub.
  6. Sync Local Repository:
    git checkout main
    git pull origin main

Handling Merge Conflicts

  1. Update Main:
    git checkout main
    git pull origin main
  2. Update Feature Branch:
    git checkout new-feature
    git pull origin new-feature
  3. Merge Main into Feature:
    git merge main
  4. Resolve Conflicts: Edit files, then
    git add .
    git commit -m "resolve conflicts"
  5. Push:
    git push origin new-feature

8. Best Practices

  • Write clear, descriptive commit messages.
  • Use branches to isolate features and fixes.
  • Regularly pull updates to stay in sync.
  • Use .gitignore to exclude unnecessary files.
  • Review pull requests carefully before merging.
  • Avoid force pushes unless necessary.
